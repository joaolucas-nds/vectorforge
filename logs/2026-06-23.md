# Log — 2026-06-23

## Objetivo da sessão
Avaliar a viabilidade e implementar do zero uma ferramenta de geração procedural de arte vetorial no browser, exportável como SVG para uso em ferramentas de design (Figma, Inkscape, Canva, CorelDraw, Affinity). O usuário trouxe a ideia como brainstorm aberto: estilos artísticos, geração por cálculo, opção de pixel art, canvas com esboço livre, caixa de texto → forma.

## Feito
- **`vectorforge.html`** criado do zero (~80KB): aplicação completa em arquivo único, zero dependências de runtime.
- 8 estilos de arte procedural implementados com algoritmos próprios: Art Deco, Baroque, Geometric (Mandala), Victorian, Celtic, Islamic, Minimal, Organic.
- 6 tipos de saída por estilo (48 combinações): medallion, frame, divider, corner/knot/star/arabesque/leaf/glyph, pattern/tessellation, ornament/cartouche/swag/acanthus/wreath/spiral/branch/flower/wave.
- Sistema de PRNG seeded (`mkRand`) para reproducibilidade total.
- 5 sliders de parâmetro: Complexity, Symmetry, Stroke width, Fill opacity, Seed.
- 8 paletas de cor coordenadas (stroke + fill + bg + fill2).
- Canvas size customizável com presets (1:1, 4:3, 16:9, 3:4, A4).
- Feature Text → Form: textarea + seleção de tipografia (5 famílias) → glyph SVG decorado.
- 4 variações automáticas com seeds offset (+0/+111/+222/+333).
- Export: SVG com declaração XML, PNG @2x via Canvas API, Copy SVG code.
- Modo Pixel Art: gerador pixel-by-pixel com simetria radial e setor angular.
- Modo Draw freehand: esboço no canvas → bounding box → ornamento posicionado.
- Controles de canvas: zoom, fit, toggle bg claro/escuro, grid de referência SVG.
- UI dark theme: DM Mono + Fraunces, accent `#c8a96e`.
- Kit de Contexto Universal gerado e preenchido: CONTEXT.md, STATUS.md, DECISIONS.md, CHANGELOG.md, IDEAS.md, ROADMAP.md, GLOSSARY.md, HISTORICO.md, LOG-TEMPLATE.md, logs/2026-06-23.md.

## Decisões
- **DEC-001**: arquivo HTML único sem build — portabilidade máxima, zero setup. Ver DECISIONS.md.
- **DEC-002**: geradores são funções puras que retornam string SVG — desacoplamento total do DOM, reutilizáveis para thumbnails.
- **DEC-003**: PRNG seeded (xorshift multiplicativo) — reproducibilidade, compartilhamento de seeds, Variations determinísticas.
- **DEC-004**: `outerHTML` replacement para atualizar canvas SVG — necessário para trocar `viewBox` junto com conteúdo; exige re-query pós-replace via `setTimeout`.
- **DEC-005**: modo Draw é aditivo (append `<g>`) — preserva composições; sem undo por ora.

## Bugs
- **Export PNG no modo Pixel Art**: `xPNG()` lê `#main-svg` em vez de `#px-canvas.toBlob()`; no modo pixel, exporta o SVG anterior. Não foi corrigido nesta sessão — registrado no STATUS.md e Backlog.
- **Botão Regenerate não sorteia seed**: chama `doGen()` com seed atual → idempotente. Fix trivial (uma linha); deixado para próxima sessão.
- Nenhum bug virou FIX-N (nenhum foi resolvido nesta sessão, só identificados).

## Aprendizados / armadilhas
- **Referência a `#main-svg` cacheada**: `outerHTML` replacement invalida qualquer referência ao elemento antigo. Sempre re-query após `doGen()`. → Adicionado como Armadilha 1 no CONTEXT.md.
- **`fitToView()` após replace**: precisa do `setTimeout`; chamar síncrono retorna dimensões do elemento removido. → Adicionado como Armadilha 2.
- **Export PNG × modo Pixel Art**: os dois modos compartilham o mesmo botão de export, mas usam elementos DOM diferentes. → Armadilha 3 no CONTEXT.md.
- **`TYPES_BY_STYLE` é obrigatório para todo novo estilo**: `buildTypeChips()` consulta essa tabela; omitir quebra silenciosamente. → Armadilha 4 + guia no HISTORICO.md.
